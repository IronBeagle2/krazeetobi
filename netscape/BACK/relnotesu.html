OVERVIEW OF NETCASTER

Netcaster: Integrated and Easy to Use

Netcaster is tightly integrated with all Communicator components, providing seamless access from the desktop to the most up-to-date information users need.  Netcaster leverages the "point-and-click" ease of use pioneered by Navigator to offer quick access to favorite channels and offline resources.
  • Integrated with Communicator. Like all other components, simply select from the Communicator menu to access Netcaster.
  • Full-screen experience. The Netcaster interface slides away discreetly to provide more screen real estate, making content and information, rather than menus and toolbars, the focus of attention. Netcaster also supports full-screen channels for the most dynamic and immersive experience. In addition, users can configure the Netcaster user interface to slide from either the right or left side of the screen to accommodate individual preferences and desktop environments.
  • Channels and Channel Finder. Netcaster's easy-to-use interface provides quick access to a user's favorite channels and offline sites. Netcaster prominently features Channel Finder, the source for the most dynamic channels on the Internet, on the user interface.
  • Customizable subscriptions. Users can add or subscribe to a channel and specify preferences as to cache size, the number of pages to download, and when and how often the content should be automatically updated. With Netcaster, users receive only the information they want.
  • Webtop mode. Users can anchor their favorite channel in webtop mode and have it automatically launch upon startup. Webtop mode provides ever-present links to important data and allows users to monitor information delivered in the background while working on day-to-day tasks and productivity applications.

KNOWN ISSUES IN THIS RELEASE

  • Netcaster is not bundled with Communicator and must be installed separately from a jar file which can be obtained from the Netscape homepage.
  • You must exit Communicator after installing Netcaster and then re-launch Communicator in order to start Netcaster successfully.
  • Netcaster is ready for use after "My Channels" appears in the selector.
  • If controls do not appear on a webtop click the tab to expose the selector controls.
  • If no dialogs appear after clicking on print, check with your system administrator that printers have been setup correctly.
  •  Closing (as opposed to Exiting) the Communicator window from which Netcaster was launched results in Netcaster also shutting down.  Additional Communicator windows can be open and closed without affecting Netcaster.
  • After starting Netcaster, there is a significant delay before the Netcaster window appears.
  • It is required that you use Communicator 4.03 with this version of Netcaster.
  • The order of the channels in the selector may change after exiting Netcaster.
  • The Automatically Load Images preference (Preferences, Advanced from the Edit menu) should be checked.
  • If you specify an invalid URL when adding a channel, the channel is added under My Channels, but does not appear when you select it. A green light may also appear; ignore this light.
  • If you change a webtop to a channel in Channel Properties, the webtop remains on your screen. Close the webtop and when you select it again in My Channels, it appears as a channel in a Navigator window.
  • If you change your Communicator preferences, such as color or font, while Netcaster is open, a system halt may occur. Change your Communicator preferences before starting Netcaster.
  • The buttons in the Netcaster window may disappear depending on the font you select.
  • Select a channel before modifying any security information in the Options dialog.
  • Your channel settings may not be preserved if Communicator terminates abnormally.
  • If you change your screen's resolution while Netcaster is running, the Netcaster window may no longer be visible.
  • The display and cache size of the Castanet channels cannot be modified.
  • If you add a Castanet channel by specifying its IP address and then access the channel again by specifying its DNS name, Netcaster will attempt to resubscribe to the channel.
  • Unless you specifically use the Update Now command to update a channel, at most, one channel will update.
  • Some of the channel preferences may not be visible until you restart Netcaster.
  • If you add two channels with the same name, an error may occur.
  • You cannot move a webtop, as indicated in the Netcaster NetHelp.
  • When Netcaster is started, two Java security dialogs may appear. If a button marked "Grant" does not appear in the dialog,  exit Communicator or Navigator and restart.

Installation Instructions

Normally Netcaster can be installed through the web, by visiting the appropriate link off the Netscape home page. In some cases you may receive a Netcaster jar file along with Communicator or Navigator.  If this is the case, follow these steps:
  1. Place the Netcaster jar file in a directory of your choice. For example, assuming nc10unb1.jar is the name of the Netcaster jar file:
    2. cp nc10unb1.jar /usr/myself/install
  2. Launch Communicator or Navigator and visit the installation directory url at:
    3. file:/usr/myself/install
  3. Click the link to nc10unb1.jar which appears in the browser window.
  4. Several dialog boxes appear that lead you through the installation process.
  5. When installation is complete you must exit Communicator/Navigator and restart the program before using Netcaster.
  6. The Netcaster menu item should now be available. You can now run Netcaster by selecting it.
  7. After installation, the jar file and the install directory are no longer needed and can be deleted or archived in case you want to reinstall in the future.
