KNOWN ISSUES IN THIS RELEASE

    General and Setup

    • This release has not been fully optimized for size or performance. 16 MB of RAM or more is recommended.
    • If you install the 16-bit version of Communicator PR2 (Windows 3.1 or 3.11) in the same directory as previous versions of Communicator, the previous versions are deleted. If you want to switch between versions while evaluating the PR2 version, install Communicator PR2 in a new directory. If you install in the same directory, do not run the deleted version's Deinstaller program. You might have to re-install Communicator PR2.
    • If you upgrade from Navigator 3.0 and select the Move Files options during Profile Setup, copy the cookies.txt file in your new user directory to your existing Navigator 3.x directory if you want the options to work with Navigator 3.x. If you choose the Copy Files option, copy the cookies.txt file from Navigator 3.x directory to your new user directory if you want the options to work with Communicator PR2. The default location for your new user directory is \Program Files\Netscape\Users\<user name>.
    • Help information is available on some, but not all, tasks and topics. Minimum Install includes Help on some functions that are not provided with the minimum installation. Resizing, maximizing, or restoring the Help window causes the contents to forward a page.
    • Plug-ins installed in previous versions of Navigator are not automatically transferred to Communicator. Re-install these plug-ins.
    • During installation, Communicator copies the address book from Navigator 3.0 to your user directory, but in order to update the address book format, import the address book file. To import address books, choose Import from the File menu in the Address Book window. The address book is the file address.htm in the user directory (the default is \Program Files\Netscape\Users\<user name>).
    • Several printing problems exist, including incorrect printing of images and improper alignment on the page.
    • On some intranets, the default Collabra server is a secure Collabra server. The setup wizard does not have any way of specifying the default discussion host as a secure Collabra server.

    Navigator

    • The Find in Page feature does not always find all the words being searched.
    • Resizing the window of a page containing Layers might cause your system to halt.
    • Resizing the window of a page containing plug-ins, forms, or style sheets causes the plug-in, form, or style sheet to lose its original position or formatting.
    • Resizing or reloading frame pages might cause the window to be redrawn incorrectly.
    • Some web counters are not displayed correctly in Communicator.
    • When trying to click on regions within Image Maps, the hot spots for clicking are shifted.
    • In Windows 3.1, resizing the window of a page containing tables that have different alignment might cause your system to crash.

      • Java and JavaScript

      • JavaScript is Off by default after installing Communicator on Windows 3.1. Enable Javascript under the Advance section of your Preferences if you have pages that make use of Javascript, or if you have pages with Style sheets.
      • The View Source command on Windows 95 and Windows NT shows the JavaScript code not dynamically created HTML. 
      • JavaScript encounters errors when performing user or group management.
      • Running Java in some circumstances might result in lost resources.
      • Maximizing and minimizing a page running an applet might cause the system to hang.
      • Java might be slower in this release than it was in previous versions of Navigator.
      • Reloading IIOP applets leaves the the server in an undetermined state.
      • In JavaScript event handlers might be lost when resizing a frames page. 
      • JsObject.getWindow requires an applet as an argument.
      • JavaScript sometimes shows the incorrect color or halts on frame pages.
      • Launching any page containing Java applets for the first time might cause a JavaScript Error.
      • toLocaleString() method in Date Class throws security exceptions.
      • Image onAbort event handler does not work correctly.
      • onUnload does not work correctly in nested frames.
      • History.back, history.forward, and history.go(n) display the local frame history and not the previous Navigator window.
      • You must include new when calling constructor functions. For example, 
        • var size = new java.awt.Dimension (...)
      • To enable JavaScript after disabling it, you must restart Communicator.
      • The View Source and Save commands do not parse JavaScript writes to HTML.

      Layering and absolute positioning

      • Pages containing layers cannot be printed.
      • The overflow property and the height property are not implemented for style sheet based positioning.
      • Percentage values cannot be specified for the CLIP attribute on the LAYER tag.
      • Negative values for the Z-INDEX attribute on LAYER and ILAYER tags are not recognized.
      • A value of SHOW for the LAYER tag's VISIBILITY attribute is ignored.
      • Tables within layers might not obey the layer's WIDTH attribute.
      • Centered HTML in a layer might not be truncated on its right side.
      • The ALIGN attribute might not be obeyed for applets and plug-ins inside layers.
      • Resizing pages with layers might cause a system halt or erratic behavior.

    Composer

    • The Undo and Redo commands might not work
    • Composer plug-in windows open behind the Composer window. Bring them to the front using the task bar.
    • The Edit Frame command is not working correctly. The Edit Frame command edits the frame set rather than the selected frame pane.
    • Trying to use an external HTML editor or image editor does not work in this release.
    • Composer does not resolve links properly if Escape characters are contained in the URL.
    • Composer does not handle certain HTML tags correctly. You should use caution when attempting to use Composer to edit HTML documents that contain these tags: SCRIPT, SEVER, STYLE, DIV, OBJECT, ILAYER, LINK, BASE.
    • Do not use this release of Composer to publish to a secure HTTP server. Composer might hang after displaying the "New Site Certificate" dialog.

How to Use MESSENGER AND COLLABRA 

  • Add a Collabra/news server and subscribe to discussion groups - use the Join Discussions from the File menu in the Netscape Message Center folder window. For more information on how to search for and subscribe to discussion groups, choose Help Contents from the Help menu.
  • New windows:
    • Folder window: displays the Netscape Message Center, which contains lists of mail folders and discussion groups to which you subscribe
    • Message List window: displays the contents of a mail folder or discussion group, and provides a collapsible message area in which you can view a selected message
    • Message window: displays the content of a selected message
  • Reusable windows - When opening mail folders, discussion groups, and messages, Communicator reuses the Message List or Message window you currently have open, instead of opening a new window
  • One- or two-pane display in message list window - The Message List window has a collapsible message area, which you can use to view a message you select from the message list
  • New mail icon - The Inbox icon on the task bar indicates when you have new mail on your server. Opening your Inbox no longer downloads new messages. To receive new messages, click the Get New Messages button on the toolbar.
  • Mail Notification application - To launch this feature, run Netscape Mail Notification from your Communicator program folder. This notification runs minimized on the Windows 95 and Windows NT 4.0 task bar. To set up this mail notification, right mouse click on the mail notification icon in the task bar. You can check to see if you have new mail (without having Communicator loaded) or run mail directly from this icon. This mail notification program remains loaded unless you exit by using the right mouse menu. This feature does not work when your mail is set up for an IMAP4 server. 
  • Reuse Message List and Message Windows - There is an option to reuse the Message List and Message windows or open a new window when another folder, discussion or message is selected. You can set this option from Mail and Discussions Preferences, Mail and Discussions tab. The option is "Reuse thread and message window" and "Reuse message window."
  • One or Two Pane Display in Thread Window - There is an option to show the message contents from the Message List window (two panes) or to show just the threads (one pane). You can set from Mail and Discussions Preferences, in the Mail & Discussions tab. The option is "show message in thread window by default." You can also switch between one or two panes by clicking on the blue arrow toggle shown in the left side of the window below the end of the thread list. 
  • Next Unread - Next Unread spans across mail folders and  discussion groups 
  • Backtrack and Forward - Backtrack is a feature, available from the Go menu, that takes you to the messages that you have just read. Forward, available from the Go menu, takes you to the messages you've backtracked from. 
  • Inbox Icon in the Netscape Task bar - Unlike Netscape Navigator 3.0x, displaying the Mail Inbox or the Netscape Message Center does not automatically get new messages for you. To receive new messages, click Get New Messages on the toolbar or the Message menu.
  • Filing or Copying Messages - Instead of trying to display both the Message window and a Message List window side by side to drag and drop messages into folders, you can file (move) or copy messages directly from the Message List window or from the Message window. Use the File toolbar button to move messages into folders or use the commands from the Message window. You can also file discussion group messages into folders that you create to capture information you want to refer to later. 
  • "More" feature - You can press the spacebar to read through all your unread messages. 
  • Send Later - You can defer sending of your messages by choosing the Send Later command under the Message menu. Choosing Send Later places your message into the Outbox folder. You can send the outgoing messages by choosing Message | Send in Outbox from the Netscape Message Center or from the Outbox's Message List window when you are online. 
  • Save Drafts - You can save messages that you are in the process of composing. From within the Composition window, click the Save Draft toolbar button. You can continue composing your message and then send it. Or, you can close the Composition window and then go into the Drafts folder and select the draft you saved. Double-click the draft and the Composition window appears for you to modify the message and send it. 
  • Filters - Filter support allow you to organize your incoming messages. Use Mail Filter Rules in the Edit menu to create filters. 
  • Views - You can display five different views in the Thread window. They are: New, Threads with New, Watched, Ignored, and All. Use the Messages command in the View menu to change views to one of these options:
      • Threads with New - Only unread messages appear. If your messages are sorted by thread and the unread message is a reply, the "parent" of this reply also displays. 
      Watched - Only new messages which belong to a thread that is marked "watched" (using Watch Thread from the Message menu) appears in this view. For example: When you only have time to read new messages for the threads that you've marked watched, you can access those threads directly by choosing the Watched view after getting new mail or opening a discussion group. 
      Ignored - All messages, including messages which belong to a thread that is marked "ignored" (Ignore Thread command in the Message menu)  appear in this view. 
      All - All messages, except ignored threads, appear in this view. 
      Note: When switching views, you may not see any changes because the views are cached. If this occurs, use the "All" view and select another message to clear the cache or close and reopen the folder or discussion group. 
      New - Only unread messages appear.
  • Showing More Columns in the Thread Window - Communicator defaults to showing a specified set of columns in the Netscape Message Center and the Netscape Message List window. You can increase or decrease the number of columns shown by clicking on the < or > buttons at the right of the column headings.
